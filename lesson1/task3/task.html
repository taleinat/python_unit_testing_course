For each test (i.e. each test method), the results will always be one of the following:

  1. Success, which means everything worked as expected.
  2. Failure, which means one of the assertions in the test failed.
  3. Error, which means an uncaught exception was raised during the test.

There is no explicit way to make a test succeed. Instead, a test's success depends on the absence of any failures or errors.

Your objective: Implement the three test methods, so that the first succeeds, the second fails and the third results in an error.
