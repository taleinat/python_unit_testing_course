So far we've been using self.assertEqual() in our tests to check expected results. There are many other such assertion methods available. Let's explore them!

The most basic assertion method is assertTrue(). It is called with a single argument and simply checks whether it is true. (It actually checks whether a value is "truthy", i.e. considered true in a boolean context. See details below.)

It is possible to write tests using only assertTrue(). For example, instead of using assertEqual(a, b), we could use assertTrue(a == b). However, using more specific assertions brings many benefits:

  • More detailed information upon test failure
  • Improved test code readability
  • Non-trivial assertions save time and effort

In this lesson, we'll explore many of the different assertions available. Note that you can always look through the full list of available assertion methods .

What does "truthy" mean? In Python, many values are considered either true or false when in a boolean context, e.g. when used as the condition of an "if" statement. For example, the integer zero is considered false, while all other integers are considered true. Most containers are considered false when empty and true when non-empty. You can check whether a value is "truthy" by passing it to the bool() function. For example, bool([1,2,3]) == True, and bool([]) == False.
